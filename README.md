# Task Manager

Веб-приложение для управления задачами. Пользователь может создавать задачи, просматривать их список, изменять статус выполнения и удалять задачи. Приложение состоит из двух частей:

- **Backend**: на Python с использованием FastAPI и PostgreSQL.
- **Frontend**: на React с использованием Vite.

---

## Используемые библиотеки и технологии

### Backend
- **Python 3.11+**
- **FastAPI** — для создания API
- **SQLAlchemy** — ORM для работы с PostgreSQL
- **Alembic** — для миграций базы данных
- **Uvicorn** — ASGI сервер

### Frontend
- **React 19** — библиотека для UI
- **Vite** — сборщик и dev сервер
- **Axios** — для HTTP-запросов к backend
- **Docker** — для контейнеризации frontend и backend

---

## Установка и запуск проекта

**Клонируйте репозиторий**
```bash
git clone <репозиторий>
cd <папка проекта>
```

Перед запуском необходимо в корне проекта создать файл .env на основе .env.example и указать необходимые переменные 
окружения.

**Запуск сборки контейнеров с помощью Docker Compose**
```bash
docker-compose up -d
```

**Перейти внутрь контейнера backend и применить миграции**
```bash
docker-compose exec backend bash
alembic upgrade head
```

Frontend будет доступен на порту 80, backend на 8080.

---

## Доступные действия на сайте

### Клиент (frontend)

- просмотр списка задач в виде таблицы;
- создание новой задачи через модальное окно;
- изменение статуса задачи (выполнена/не выполнена);
- удаление задачи;
- просмотр полного текста длинного описания через кнопку «▼»;
- получение уведомлений об ошибках (например, создание задачи с уже существующим названием).

### Сервер (backend)

- GET /tasks/ — получение списка всех задач;
- POST /tasks/ — создание новой задачи;
- PATCH /tasks/{task_id} — изменение статуса задачи;
- DELETE /tasks/{task_id} — удаление задачи.

Для удобного изучения API можно использовать Redoc и Swagger документацию, которая доступна сразу после запуска backend:
- Redoc: `http://<backend_host>:8080/redoc`
- Swagger: `http://<backend_host>:8080/docs`
